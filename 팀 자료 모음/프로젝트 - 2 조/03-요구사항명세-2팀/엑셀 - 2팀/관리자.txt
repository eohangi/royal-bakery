
회원관리

해당 관리 태그에는 별도의 리스트,태그가 없다. [회원관리 버튼 클릭시 바로 해당페이지로 이동.]
규칙:회원자료 삭제 시 다른 회원이 기존 회원아이디를 사용하지 못하도록 회원아이디, 이름, 닉네임은 삭제하지 않고 영구 보관합니다.
해당 아이디값에 기록된 주문내역도 보관.

따라서 삭제될때에는 이름에 관련된 정보 삭제.. 문제는 중복되는 이름일경우 어떻게 처리? (if문으로 생년월일찾아서 복잡하게 처리하게될듯..)

<정보제공: 회원정보 / 검색바 / 정렬 / 버튼:회원정보 수정, 회원정보 삭제 / 페이지목록>
리스트: 이름/ 아이디/ 전화번호뒷자리/ 생년월일 / 그룹 노출
정렬[드롭다운] - 이름,아이디별 (기본정렬 - 최근 가입 날짜를 기준으로 회원 리스트 노출 )
검색: 이름, 아이디, 전화번호 뒷자리 검색
해당 div 페이지 상단에는 검색파트와 목록파트, 페이징파트(페이지 번호)가 있다.
검색과 정렬을 통하여 필터링이 가능하다.
기본적으로 회원목록이 출력되고 각 아이디 목록의 오른쪽 끝에는 수정, 삭제 버튼이 있다. 
이 버튼을 누르면 해당 아이디의 상세정보가 출력되는 페이지로 이동. 페이지 하단에는 수정확인 버튼이 있다.
그룹은 일반회원과 관리자를 구분하기 위한것.
(VIP시스템은 차후에 프로젝트 완료후 도입예정 [특정 구입금액을 넘을시 자동으로  vip로 승급. -스타벅스 시스템])


정보제공: 회원정보 
수정 - 회원 정보를 수정하는 팝업창이 생성. 버튼: 수정, 취소
삭제 - 해당 아이디의 정보"만" 삭제하는 것으로 삭제버튼을 클릭시 재확인을 위한 확인 팝업창이 나온다.
